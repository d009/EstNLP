1. Tekst koos mrgendustega: Text klass

Text klass on EstNLTK keskne komponent. See sisaldab analsitavat teksti, teksti metaandmeid ning analsi kigus loodavaid lingvistilisi mrgenduskihte. Nide:

In[1]:
from estnltk import Text
text = Text('ra mine sinna, kuhu viib rada. Mine selle asemel sinna, kus pole htki rada ja ole teerajaja.')
text
Out[1]:
text
ra mine sinna, kuhu viib rada. Mine selle asemel sinna, kus pole htki rada ja ole teerajaja.

Atribuudi text kaudu saab algse teksti uuesti snena ktte:

In[2]:
text.text
Out[2]:
'ra mine sinna, kuhu viib rada. Mine selle asemel sinna, kus pole htki rada ja ole teerajaja.'

Algne tekst pole muudetav


EstNTLK disain jrgib phimtet, et algne tekst pole muudetav. Seega, kui vaja teha algses tekstis (snes) muutuseid, tuleb muudetud sne jaoks tekitada uus Text objekt.
Remark: Kuigi teksti otseselt muuta ei saa, on teatud juhtudel vimalik teha seda kaudselt, mrgenduste muutmise lbi. Niteks saab snu normaliseerida: lisada mittekirjakeelsele snale kirjakeelseid normaliseeringuid -- nii, et lingvistiline anals ei toetu mitte mittekirjakeelsele vormile, vaid normaliseeritud snavariantidele. Selle kohta vt lhemalt siit.

1.1 Teksti metaandmed

Teksti metaandmed on salvestatud snastiku kujul ning neile pseb ligi atribuudi meta kaudu. Nide:

# metaandmete omistamine
text.meta = {'author': 'Tundmatu', 'date': 2015}
# metaandmete kshaaval lisamine
text.meta['origin'] = 'tsitaadid.ee'
text.meta['url'] = 'https://tsitaadid.ee/quote/576/14'

Vaikimisi pole loodaval Text objektil metaandmeid -- need tuleb lisada kasutajal. Siiski korpuse importimise funktsioonid tagastavad tekstid koos metaandmetega (kui korpuses on metaandmed olemas).

Kui on kavas andmeid serialiseerida / salvestada Postgres andmebaasi, siis peavad meta vrtused kasutama ainult andmetpe str, int, float ja datetime.

1.2 Mrgenduskihtide lisamine ja eemaldamine

Mrgenduskiht ( layer ) koondab endas tekstis ramrgitud asukohti ( span ) ning neile lisatud mrgendusinfot ( annotations ). Iga kiht fikseerib kindlad atribuudid, millega mrgendusinfot edasi antakse.

1.2.1 Meetod tag_layer

Meetodi tag_layer abil lisatakse Text-ile mrgenduskihid, rakendades selleks EstNLTK tahelas olevaid mrgendajaid. Nide:

In[3]:
text.tag_layer(['tokens', 'words'])
Out[3]:
text

layer name attributes parent enveloping ambiguous span count
tokens None None False 21
compound_tokens type, normalized None tokens False 0
words normalized_form None None True 21

Kui mrgenduskiht 'tokens' vlja arvata, siis enamik teisi kihte omab sltuvusi: neid ei saa tekitada enne, kui on loodud sltuvuskihid. Meetod tag_layer haldab sltuvusi automaatselt ja tekitab kik vajaminevad kihid. Eelmises nites: lisaks 'tokens' ja 'words' kihile tekitati ka 'compound_tokens' kiht, kuna 'words' kihi loomine nuab seda.

Kui tag_layer kutsutakse vlja ilma kihte tpsustava sisendargumendita, siis on vaikevrtuseks ['morph_analysis', 'sentences'] -- tekivad vastavad kihid koos nende sltuvustega:

In[4]:
text.tag_layer()
Out[4]:
text

sentences None words False 2
tokens None None False 21

words normalized_form None None True 21
morph_analysis normalized_text, lemma, root, root_tokens, ending, clitic, form, partofspeech words None True 21

NB! Kui kutsuda tag_layer vlja tekstil, kus on juba nutud kihid olemas, siis kihtide uuendamist vi lekirjutamist ei toimu. Kui on soov kihti uuendada (nt teostada morfoloogilist analsi teiste stetega), siis tuleb kigepealt vana kiht eemaldada (vt allpool) ning alles seejrel mrgendada kiht uuesti.

Mida tag_layer tagastab? Tagastatavaks vrtuseks on see sama Text objekt, millel tag_layer vlja kutsuti:

In[5]:
new_text = text.tag_layer()
assert text == new_text

1.2.3 Mrgendajate importimine estnltk.taggers kaudu. Mrgendajate otserakendamine (meetod tag)

DEFAULT_RESOLVER ei sisalda kiki EstNLTK-s olemasolevaid mrgendajaid. Phjus selles, et osad mrgendajad on rakendamiseks spetsiifilistest kontekstides (nt mrgenduste vrdlemisel), osad mrgendajad nuavad spetsiifilisi ressursse (nt suuri mudeleid, mis tuleb veebist alla tmmata) ja osad tegelevad spetsiifiliste tekstianalsi probleemidega (nt kuupevade tuvastamisega meditsiinitekstides). Mrgendajad (Tagger) vs mbermrgendajad (Retagger)


Osad EstNTLK mrgendajad loovad uusi kihte, osad aga kirjutavad mber (parendavad) olemasolevaid kihte. Kui tegemist on kihti mberkirjutava mrgendajaga (Retagger), siis tuleb enne selle rakendamist veenduda, et mberkirjutatav kiht (output_layer) on tekstil juba olemas ning mrgendaja rakendamine kib siis meetodi retag( text ) kaudu.

Mrgenduskihi nime muutmine


Kui importida mrgendaja estnltk.taggers-i kaudu, siis saab ka muuta loodava mrgenduskihi nime. See kib tavaliselt konstruktori argumendi output_layer abil, nt verbiahelate tuvastaja puhul: 
from estnltk.taggers import VerbChainDetector
verb_chain_tagger = VerbChainDetector(output_layer='my_verb_chains')
Mrgenduskihitide nimede muutmine on vajalik triistade erinevate konfiguratsioonide testimisel, samuti triistade tiustamisel. Nt, kui nimetada mrgenduskihid vastavalt triista versioonile (a la 'verb_chains_v1', 'verb_chains_v2' jne), siis saab neid ka vrrelda omavahel.

1.2.4 Tahelas olevate mrgendajate muutmine (make_resolver)

EstNLTK baastahela tekitamine (mrgendajate initsialiseerimine ja lisamine ahelasse) kib make_resolver-i abil.

Kige lihtsam viis tahela muutmiseks on luua make_resolver-iga baastahelast koopia ja muuta seda vastavalt oma vajadustele. Nide:

In[15]:
from estnltk.default_resolver import make_resolver

my_resolver = make_resolver()  # Loome baastahela koopia

Seejrel saab meetodi update abil uuendada tahela mrgendajat:

In[16]:
# Loome uue morf mrgendaja, millel on hestamine ja oletamised vlja llitatud
from estnltk.taggers import VabamorfTagger
vabamorf_tagger = VabamorfTagger( disambiguate=False, guess=False, propername=False )

# Asendame tahelas oleva morf mrgendaja uuega
my_resolver.update( vabamorf_tagger )

Muudetud tahela rakendamiseks tuleb meetodi tag_layer vljakutsumisel tpsustada, millist resolver-it kasutame:

from estnltk import Text
text = Text('Metsawahi hobusele om uus laut ehitet.')
text.tag_layer('morph_analysis', resolver=my_resolver)

LayerResolver-i meetodi update(tagger) abil saab tahelasse lisada ka tiesti uusi mrgendajaid, aga tingimuseks on, et lisatava mrgendaja kiki sltuvuskihte ( input layers ) peab tahel oskama juba teha.

make_resolver ja morfoloogilise analsi parameetrid


Kuna EstNTLK lingvistilise analsi keskne komponent on morfoloogiline mrgendaja, siis on vimalik make_resolver-i abil ka otse muuta morfoloogilise analsi parameetreid disambiguate, guess, propername jm. Selle kohta on detailsemalt juttu juhendmaterjalis introduction_to_nlp_pipeline.ipynb.

Hoiatus: make_resolver ja konfliktsed konfiguratsioonid


make_resolver ja DEFAULT_RESOLVER pakuvad morfoloogilise analsi vaikekonfiguratsiooni, millega toimivad ka teised tahelas olevad komponendid. Kui aga muuta make_resolver-i abil morfoloogilise analsi parameetreid, ei garanteeritud, et silib tahela analsi kvaliteet ning kigi komponentide tvime. Nt kui llitada lalkirjeldatud viisil vlja hestamine ja oletamine, siis ei toimi enam nimeksuste tuvastamine ja sntaktiline anals. Seega tasub tahela konfiguratsiooni muuta alles siis, kui on selge, kuidas see erinevaid triistu mjutab.

Kui oled kogemata muutnud vaiketahelat selliselt, et osad komponendid enam ei toimi, siis saad vaiketahela taastada make_resolver-i abil:

In[18]:
from estnltk.default_resolver import make_resolver
# Taastame tahela vaikekonfiguratsiooni
Text.layer_resolver = make_resolver()

Mrgenduste asukohaphine vrdlemine (estnltk_core.layer.span_operations)


EstNLTK-s on olemas ka funktsioonid Span-ide sstemaatiliseks asukohaphiseks vrdlemiseks. Niteks:

Vrdlusfunktsioone on veel, kigi funktsioonide kohta vaata detailsemalt lhtekoodist: https://github.com/estnltk/estnltk/blob/main/estnltk_core/estnltk_core/layer/span_operations.py

Layer objekti loomisel saab tpsustada jrgmised parameetrid:

  • name -- kihi nimi (sne). Vlistatud kihinimed: text ning kihinimed, mida kasutavad EstNLTK tahela mrgendajad (vt tpsemalt: 1.2.2. Millised kihte saab tag_layer abil tekitada? DEFAULT_RESOLVER );

  • attributes -- jrjend kihi mrgenduste atribuutide nimedega. Need atribuudid saavad olema kigil kihi Annotation objektidel. Vlistatud atribuudinimed: start, end ja text;

  • text_object -- kihiga seotud Text objekt. Kui Text objekt on sidumata, pole vimalik uurida mrgenduste tekstisisu (text ja enclosing_text);

  • parent -- lemkihi nimi (sne). Kui lemkiht puudub (ning pole tegu mbriskihiga), lisatakse mrgendused otse algtekstile, aga lemkihi olemasolul saab mrgendusi lisada ainult lemkihi mrgendustega samadele asukohtadele;

  • enveloping -- mbritsetava kihi nimi (sne). Kui mbritsetav kiht on mratud, siis tuleb mrgenduste lisamisel tpsustada jrjend mbritsetava kihi mrgenduste asukohtadest, mida antud kiht katab;

  • ambiguous -- kas kiht on mitmene (tevrtus). Kui kiht on mitmene, siis saab he asukohaga siduda mitu Annotation objekti;

  • default_values -- mrgenduse atribuutide vikevrtused (snastik).

  • secondary_attributes -- jrjend kihi atribuutidest, mida ei arvestata kihtide/mrgenduste vrdlemisel. secondary_attributes peab olema attributes alamosa.

2.1.5 Kokkuvtvalt: kihi mrgenduste lisamine, muutmine ja eemaldamine

Layer vimaldab:

  1. lisada mrgendusi:
    • layer.add_annotation( base_span ) (ilma atribuutideta kihi korral)
    • layer.add_annotation( base_span, **keyword_attributes )
    • layer.add_annotation( base_span, attributes_dict )
  2. kustutada mrgendusi:
    • layer.remove_span( span: Span ) (objekti jrgi)
    • del layer[i] (indeksi jrgi)
  3. muuta mrgendusi:
    • layer[i].annotations[j][attr_k] = ... (indeksite jrgi)

Disainiphimte: mrgendusi tekitavad ja muudavad mrgendajad


EstNLTK jrgib disainiphimtet, et mrgenduskihtide loomine ja muutmine toimub mrgendajate sees. See tagab, et loodavad kihid on valiidse struktuuriga, kuna mrgendajate sees toimub kihtide jrelkontroll. Mrgendajate loomise ja kasutamise kohta vt detailsemalt ptk 2.2.

Mrgenduste lisamise ja muutmine Span / EnvelopingSpan abil


Mrgendusi on vimalik muuta ja lisada ka Span ja EnvelopingSpan objektidega opereerides, aga see on veidi keerukam ning uut kihti luues tasub jda Layer-i poolt pakutava liidese juurde. Kui on aga vaja tekitada kihti muutev Retagger, vib lisateadmine Span ja EnvelopingSpan liidesest olla kasulik. Nimelt, olemasoleva kihi mrgendusi saab muuta Span ja EnvelopingSpan kaudu jrgmiselt:
  1. eemalda (kik) mrgendused: span.clear_annotations();
  2. lisa uus mrgendus: span.add_annotation( annotation_dict );
  3. muuda i-nda mrgenduse attr_j vrtust: span.annotations[i][attr_j] = ...;

Mrgenduste lisamise, muutmise ja eemaldamise reeglid


  • Kihil ei tohi olla mitut tpselt hesuguse asukoha ehk Span-iga (EnvelopingSpan-iga) mrgendust (aga osaliselt kattuva asukohaga mrgendused on lubatud);
  • Iga kihil olev Span / EnvelopingSpan peab sisaldama vhemalt ht Annotation objekti (kuigi see vib olla ka thi, nagu snestusmrgenduse puhul);
  • Mistahes liiki kihile ei saa lisada mitut tpselt samade atribuudivrtustega Annotation-it -- duplikaatide puhul jetakse alles vaid ks Annotation;

2.2 Mrgendajad / mbermrgendajad

2.2.1 Tagger (kihi looja)

Mrgendaja abil luuakse uusi kihte. Mrgendaja tegemine samm-haaval:

  1. Loo estnltk.taggers.Tagger klassi alamklass;
  2. Lisa kik mrgendaja konfiguratsiooni parameetrid klassimuutujasse conf_param (muutuja tp Sequence[str]);
    Ainult selles loendis olevaid parameetreid tohib mrgendaja alla salvestada. Kui parameetri nime ees on alakriips (_), siis loetakse see sisemiseks parameetriks ning mrgendaja konfiguratsiooni kuvamisel seda ei nidata;
  3. Lisa vljundkihi nimi klassimuutujasse output_layer(tp str);
  4. Lisa kik vljundkihi atribuutide nimed muutujasse output_attributes(tp Sequence[str]);
  5. Lisa kik kihid, mida on vaja vljundkihi loomiseks, muutujasse input_layers(tp Sequence[str]);
  6. Loo konstruktor __init__, kus pannakse lplikult paika parameetrite conf_param, output_layer, output_attributes, input_layers ning teiste konfiguratsiooniparameetrite vrtused. Mrgendaja konfiguratsioon peaks olema tielikult mratud konstruktoris, vljaspool konstruktorit selle muutmist toimuda ei tohiks. (Kui kasutajal on vaja muuta konfiguratsiooni, peaks ta tekitada uue mrgendaja uue konfiguratsiooniga);
  7. Loo meetod _make_layer_template(self) -> Layer, mis tekitab thja eraldatud kihi, kus on kik kihi atribuudid seadistatud vastavalt mrgendaja konfiguratsioonile;
  8. Loo meetod _make_layer(self, raw_text: str, layers: Mapping[str, Layer], status: dict=None) -> Layer, milles tekitatakse uus Layer objekt (soovitatavalt _make_layer_template() abil), tidetakse see andmetega ning tagastatakse;

Lihtsustatud nide: loome mrgendaja, mis mrgendab toiduretseptides koguseid (nt 1 tl , 200 g , 2 dl ):

In[66]:
from estnltk import Text, Layer
from estnltk.taggers import Tagger

class QuantityTokensTagger(Tagger):
    """Tags tokens that make up quantity expressions.""" 
    conf_param = ['quantity_lemmas']
    
    def __init__(self, # output_layer name can be changed:
                       output_layer='quantity_tokens',
                       # input layer name can be changed:
                       input_morph_analysis_layer='morph_analysis',
                       # quantity lemmas can be changed:
                       quantity_lemmas=['tk', 'tl', 'dl', 'kg', 'g']):
        # Set input/output layers
        self.input_layers = [input_morph_analysis_layer]
        self.output_layer = output_layer
        self.output_attributes = ['token_type']
        # Set other configuration parameters
        self.quantity_lemmas = set(quantity_lemmas)

    def _make_layer_template(self):
        # Create new detached layer debased on the configuration
        return Layer(name=self.output_layer, attributes=self.output_attributes, text_object=None)
        
    def _make_layer(self, text, layers, status):
        # Create new layer based on the configuration
        layer = self._make_layer_template()
        # Assign the Text object
        layer.text_object = text
        for span in layers[ self.input_layers[0] ]: # Iterate over 'morph_analysis' (first input layer)
            for annotation in span.annotations:
                if annotation['lemma'] in self.quantity_lemmas:
                    # Mark units
                    layer.add_annotation(span.base_span, token_type='UNIT')
                    break
                if annotation['lemma'].replace('.','',1).isdigit():
                    # Mark numbers
                    layer.add_annotation(span.base_span, token_type='NUMBER')
                    break
        # Return created layer
        return layer

Rohkem detaile


  • Sisuliselt on Tagger-il neli meetodit, mis haldavad kihi loomist:
    • tag(text: Text, status: dict) -- luuakse ja lisatakse kiht etteantud Text objektile. Lppkasutajale meldud meetod, mida mrgendajate arendajad reeglina muuta / le kirjutada ei tohiks;

  • Muudetavad kihtide nimed: on soovitatav, et sisend- ja vljundkihtide nimed oleksid mrgendajas muudetavad: 1) konstruktori parameetrite abil peaks olema vimalik muuta input_layers ja output_layer vrtuseid; 2) meetodis _make_layer(...) peaks eksplitsiitsete kihinimede asemel kasutada kihinimede muutujaid input_layers ja output_layer. 2.2.2 Retagger (kihi muutja)

mbermrgendaja muudab / parandab olemasolevat kihti. mbermrgendaja on Retagger klassi alamklass, millel on mratud atribuudid:

  • conf_param -- list legaalsetest parameetrinimedest (snede list);
  • output_layer -- muudetava kihi nimi (sne);
  • output_attributes -- list muudetava kihi atribuutide nimedega (snede list);
  • input_layers -- list muutmiseks vajalikest sisendkihtidest (snede list); NB! peaks sisaldama ka kihti output_layer;

ning implementeeritud meetodid:

  • __init__ -- konstruktor, kus pannakse lplikult paika parameetrite conf_param, output_layer, output_attributes, input_layers ning teiste legaalsete parameetrite vrtused, mis moodustavad konfiguratsiooni;
  • _change_layer(...) -- meetod, mis muudab (vastavalt mrgendaja konfiguratsioonile) kihti output_layer ning ei tagasta midagi;

Retagger-i lesehitus on suuresti analoogne Tagger-i omale. Ka Retagger-is haldavad kihi muutmist 3 meetodit: _change_layer(...) (kihi muutmise loogika implementatsioon), change_layer(...) (andmebaasiliidese jaoks vajalik meetod) ning retag(...) (lppkasutajale meldud meetod). Ent Erinevalt Tagger-ist thja kihi loomise meetodit siin implementeerima ei pea.

Lihtsustatud nide mbermrgendajast: snade normaliseerija, mis lisab spetsiifilistele vana kirjakeele snadele ( nt om ja ehitet ) normaliseeritud vormid ning mrgib iga sna puhul ra, kas see sai normaliseeritud vi mitte:

            if current_norm_forms == [None]:
                current_norm_forms = [span.text]
            # Try to replace current normalized forms with forms from the lexicon
            new_forms = []
            change_status = []
            for cur_form in current_norm_forms:
                if cur_form.lower() in self.lexicon:
                    new_forms.append(self.lexicon[cur_form.lower()])
                    change_status.append(True)
                else:
                    new_forms.append(cur_form)
                    change_status.append(False)
            # Clear existing annotations and add new ones that have 1 extra attribute
            span.clear_annotations()
            for form_id, new_form in enumerate( new_forms ):
                span.add_annotation( {'normalized_form': new_form, 
                                      'is_normalized' : change_status[form_id] } )
